# Security Policy — D00MGATE-NECH9

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x-alpha | ✅ Active |

## Reporting a Vulnerability

If you discover a security vulnerability in the **public demo layer**:

1. **DO NOT** open a public GitHub issue
2. Contact privately via GitHub: [@nDumitru](https://github.com/nDumitru)
3. Include: description, reproduction steps, potential impact
4. You will receive a response within **72 hours**

## CTF Challenges — Special Note

Vulnerabilities found **within the CTF challenge** are **intentional by design**.  
The CTF is designed to be broken at Levels 1–3.

Level 4 (Master Key) has no known solution.  
If you believe you have found one — contact privately. Immediately.

## Scope

| In Scope | Out of Scope |
|---|---|
| Demo layer bugs | Core protocol internals (not public) |
| CTF challenge logic | Master Key mechanics (not public) |
| Documentation errors | Four Races Architecture (not public) |

## Responsible Disclosure

We follow a **90-day responsible disclosure** policy.  
Researchers who report valid vulnerabilities will be credited in CHANGELOG.
